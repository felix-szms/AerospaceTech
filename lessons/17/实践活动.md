# 实践活动：海报定稿 · 视频拍摄 · 模拟答辩

> 第 17 课实践环节（约 40 分钟）｜ EDP 之 Improve（改进）阶段
> 🏆 这是赛前最后一次打磨，本课产出直接对接赛事交付。

---

## 活动目标

1. 对照赛事规范完成**展示海报定稿**，通过同伴"3 秒测试"。
2. 完成**视频脚本**并拍摄关键演示镜头（实物运行画面）。
3. 按骨架完成**答辩 PPT**，明确主讲人与分工。
4. 完成**模拟答辩**（展演 + 问答），获得教师与学生评委反馈。
5. 根据反馈制定**最终修改计划**，落实到责任人和时间点。

## Phase 1 · 海报定稿（10 分钟）

### 步骤

1. **打开**上节课起草的海报文件（Canva / PPT / AI）。
2. **对照自检清单**（见 `contest-templates/展示海报规范.md` 末尾）逐项打勾。
3. **3 秒测试**：邀请相邻小组站到 1.5 米外看你的海报 3 秒，说出他抓到的核心信息。
   - 他说对了 → 通过。
   - 他说不出"解决什么问题 / 怎么做" → 回去调标题、系统框图、创新点。
4. 导出 JPG（< 5MB，≥150 DPI），同时保存源文件。

✅ **本 Phase 产出**：海报终稿（JPG）+ 通过 3 秒测试的同伴签字。

## Phase 3 · 答辩 PPT 制作（5 分钟）

### 步骤

1. **打开** `contest-templates/答辩PPT骨架.md`，按 8–12 页骨架快速填充：
   - 封面 → 问题引入 → 现状不足 → **创意核心页** → 科学原理 → 设计与实现 → 测试数据 → 创新点 → 应用前景 → 致谢。
2. **图多字少**：每页一个核心信息，复用海报里的系统框图、结构图、数据图（省时间）。
3. **确定主讲人**与分工（谁讲哪几页、谁操作演示、谁答哪类问题）。
4. 存为 PDF 备份。

✅ **本 Phase 产出**：PPT 初稿（8–12 页）+ 主讲人确定 + PDF 备份。

## Phase 4 · 模拟答辩（15 分钟）

> 🏆 这是本课的核心环节。完全按国赛"作品展演 + 问答"流程走。

### 答辩流程（每组 5 分钟）

| 环节 | 时长 | 说明 |
|---|---|---|
| 作品展演 | **3 min** | 主讲人按黄金圈 Why–How–What 讲；可演示原型 |
| 评委问答 | **2 min** | 教师 + 学生评委各提 1–2 问 |
| 切换 | 0.5 min | 下一组就位 |

### 角色分配

- **展演组**：主讲人主讲，其余成员演示原型 + 补充问答。
- **教师**：主评委，提 1 个核心问题（从常见 7 类中选）。
- **学生评委**：每组派出 1 人担任评委，用《模拟答辩评分表》打分并提问（非本组）。
- **观摩组**：其余同学观摩学习，记录"我想学的一点"和"我会提的一个问题"。
- **计时员**：教师或指定学生，3 分钟展演到点响铃，超时扣分。

### 常见评委提问清单（教师/学生评委从中选题）

1. 你的方案和已有的 XX 产品/研究有什么区别？（创新性）
2. XX 模块的科学原理是什么？怎么实现的？（科学性）
3. 你实际做出了什么？测试数据能证明什么？（真实性）
4. 成本多少？能大规模推广吗？（可行性）
5. 经费/时间充足的话，下一步做什么？（前瞻性）
6. 团队成员各自做了什么？（团队协作）
7. 这个方案有什么风险或不足？（批判性——坦诚加分）

> 🎤 **录像**：教师用手机录制每组答辩，供 Phase 5 及课后复盘。

✅ **本 Phase 产出**：每组完成一次完整展演+问答；获得评分表反馈。

## 讨论问题（如时间允许）

1. 模拟答辩中，你觉得**最难回答**的问题是哪一类？为什么？
2. 看完其他组的展演，你们组最需要补强的是"讲清楚"还是"做出来"？
3. 如果正式答辩时紧张忘词，你的"救场预案"是什么？

---

## ⚠️ 注意

- **时间管理是本课成败关键**：教师严格控时，每组展演到点响铃，**不拖堂**。
- 模拟答辩**不追求完美**，目的是暴露问题；今天发现的问题越多，正式答辩越稳。
- 录像和评分表课后要复盘；修改计划表务必落实到责任人和期限。
- 实物演示前先**测试设备能正常工作**，避免现场翻车。
