# 展示视频制作指南

> 🏆 对接：全国青少年航天创新大赛（材料评审环节）+ 省市青少年科技创新大赛
> 时长：3–5 分钟。格式以当年赛事要求为准（通常 MP4）。

---

## 一、视频定位与作用

展示视频是**材料评审**环节评委了解作品的主要途径（在现场答辩之前）。一个好的视频能：
- 让评委在 3–5 分钟内完整理解你的方案。
- 展示实物/原型的**实际运行画面**（比文字描述有说服力 10 倍）。
- 体现团队的制作过程（真实性）。

---

## 二、视频结构（3–5 分钟脚本模板）

| 时间 | 板块 | 内容 | 画面 |
|---|---|---|---|
| 0:00–0:15 | 开场 | 标题 + 团队 | 标题动画 + 太空背景 |
| 0:15–0:45 | 问题 | 发现的真实问题 + 场景 | 场景实拍 / 新闻画面 |
| 0:45–1:15 | 创意 | 一句话说清方案 | 系统总览图 / 动画 |
| 1:15–2:00 | 原理与设计 | 科学原理 + 结构 | 原理动画 + 三维模型 |
| 2:00–3:30 | 实现与演示 | **实物/原型运行演示**（核心）| 实拍原型工作过程 |
| 3:30–4:15 | 测试数据 | 测试结果图表 | 数据可视化动画 |
| 4:15–4:45 | 创新点与应用 | 3 条创新 + 应用场景 | 文字 + 场景图 |
| 4:45–5:00 | 结尾 | 团队 + 致谢 | 团队合影 |

## 三、制作流程（与课程衔接）

### Phase 1：脚本撰写（第 16 课，1 课时）
- 写旁白脚本（每段对应画面）。
- 画分镜草图（每个镜头画一小格）。

**脚本模板**：

| 镜头 | 时长 | 旁白/字幕 | 画面 | 素材来源 |
|---|---|---|---|---|
| 1 | 3s | （标题）| 标题动画 | Canva/剪映模板 |
| 2 | 8s | "全球有2.5亿视障人群……"| 盲人行走场景 | 实拍/图库 |
| … | | | | |

### Phase 2：素材拍摄/录制（第 16–17 课之间，课后）
- **实物运行画面**（最重要）：多角度拍摄原型工作过程。
- 制作过程花絮（增强真实性）。
- 团队成员讲解镜头（可选）。
- ⚠️ 拍摄注意：光线充足、背景干净、横屏拍摄、稳定（用三脚架）。

### Phase 3：剪辑（第 17 课，1 课时 + 课后）
- 按脚本剪辑。
- 配旁白（建议录音后导入，比现场收音清晰）。
- 加字幕（重要数据/创新点用字幕强调）。
- 配背景音乐（版权免费！见下方推荐）。
- 加片头片尾。

### Phase 4：审片修改（第 17 课后）
- 对照自检清单。
- 同伴互评。
- 导出 MP4。

## 五、制作要点

### 1. 实物演示是核心
- 视频的灵魂是**展示你的原型真的能工作**。
- 多角度、多场景拍摄。
- 如果原型只实现了部分功能，**诚实展示已实现的部分**，未实现部分用动画/概念图说明。

### 2. 旁白清晰 > 画面炫酷
- 评委关心的是**方案讲清楚了没有**，而非特效。
- 旁白语速适中，口齿清楚。
- 关键信息配字幕。

### 3. 节奏
- 前 15 秒抓住注意力（问题引入）。
- 中段信息密度最高（原理+实现+测试）。
- 结尾有力（创新点+应用）。

### 4. 真实感
- 穿插**制作过程花絮**（焊接、调试、3D 打印过程）——证明是你做的。
- 出现团队成员（讲解/操作）——增强可信度和团队感。

### 5. 时长严控
- **不要超时**。赛事通常严格限制时长，超时可能被扣分或截断。
- 3 分钟能讲清就别拖到 5 分钟——精简是能力。

## 六、常见问题与改进

| 问题 | 改进 |
|---|---|
| 像产品广告，没有技术细节 | 增加原理、数据、实现过程 |
| 只有 PPT 录屏没有实物 | 必须加实物运行画面 |
| 旁白含糊听不清 | 重新录音，用麦克风 |
| 画面晃动 | 用三脚架/稳定器 |
| 背景音乐盖过人声 | 降低 BGM 音量至人声的 20–30% |
| 超时 | 删减非核心内容 |
| 字幕错别字 | 仔细校对 |

## 💡 教师指导要点

1. **第 16 课先写脚本**：磨刀不误砍柴工。脚本定了，拍摄和剪辑才有方向。
2. **拍摄趁早**：实物演示画面需要在原型完成后尽快拍，留出重拍余地。
3. **强调"真实"**：告诉学生这不是拍科幻宣传片，评委会看"你是不是真做了"——花絮和过程比特效加分。
4. **分工明确**：建议 1 人负责脚本/旁白，1 人拍摄，1 人剪辑，避免人人插手导致混乱。
5. **备份习惯**：视频文件容易丢失，要求学生云端+本地双备份。
